# 文獻 I 統計報表與資訊行為（Statistics & Information Behavior）摘要

> 說明：本文件為「摘要版」（去除匯出對話資訊），聚焦「中小學需要的最小報表」與「可用但不侵入的行為資料」；檢索細節請參考文獻 E。

## 1) 核心概念與名詞（中英文對照表）
| 術語 (Term) | 一句話定義 | 同義詞/常見誤用 |
| --- | --- | --- |
| 統計報表（Report） | 將館藏/借閱/使用情形用表格或圖表呈現 | 報表越多越好；其實要先定決策用途 |
| 指標（Metric） | 可量化的數值（借閱量、逾期量） | 沒定義計算方式就直接比較 |
| KPI（Key Performance Indicator） | 與目標直接相關的關鍵指標 | 把好看的數字當 KPI（如總藏量） |
| 館藏利用率（Use / Utilization） | 館藏被借閱/被使用的程度 | 用「借閱量」代替所有利用（館內閱覽常被忽略） |
| 流通率（Turnover） | 特定期間的借閱次數相對於館藏量 | 忽略不同類型館藏的差異（繪本 vs 參考書） |
| 零借閱（Zero-circulation） | 長期未被借閱的館藏 | 直接等同「應汰舊」，未考慮典藏/參考用途 |
| 分群（Segmentation） | 依年級/班級/類別區分統計 | 分群過細導致個人可被推回（再識別風險） |
| 匯出（Export） | 將報表輸出成 CSV/Excel 供後續分析 | 只做儀表板不做匯出，資料被鎖死 |
| 事件（Event） | 系統中可被記錄的動作（借出、歸還、搜尋） | 為了分析而記錄過多個資或明細 |
| 匿名化（Anonymization） | 移除/改寫資料使其無法對應到個人 | 只移除姓名但保留可再識別的組合欄位 |
| 瀏覽（Browse） | 以分類/分區/推薦清單等方式探索 | 只提供搜尋框，忽略小學生常靠瀏覽 |
| 搜尋行為（Search Behavior） | 查詢、點擊、改查詢等互動行為 | 把搜尋失敗都怪使用者，未改善索引與提示 |

## 2) 關鍵理論與必讀脈絡（為何重要）
- **報表服務決策**：報表要回答「我該買什麼、汰什麼、怎麼教」而非只堆數字。
- **可比較性**：指標必須定義口徑（期間、是否含館內閱覽、是否排除教師借閱等）。
- **隱私與分析的平衡**：行為資料最容易踩到個資；採「彙總優先、明細最少」。
- **兒童資訊行為**：小學生常以瀏覽、圖像、熱門清單找到書；UI 與報表都要回饋這點。

## 3) 標準/規範/權威資源（可搜尋線索）
- IFLA school library guidelines（含學校圖書館評估與服務指標）搜尋：`IFLA school library evaluation indicators`
- ALA/學校圖書館協會關於閱讀推廣與利用評估（搜尋：`school library circulation statistics evaluation`）
- 隱私與分析（搜尋：`library analytics privacy guidelines`）

## 4) 中小學場域的簡化策略（做/不做/為什麼）
- **先做 6 份報表就夠**：熱門書、逾期清單、班級借閱、分類借閱、零借閱、館藏異常（遺失/破損）。
- **先做匯出，不急著做大儀表板**：Excel/Google Sheets 即可做二次分析，成本低。
- **彙總優先**：學生層級的排行/明細預設不提供（除非館員依法有需要且可控）。
- **分群避免過細**：年級層級通常足夠；班級視學校政策與隱私評估決定。

## 5) 對應到系統的資料欄位建議（至少 8 欄）
| 欄位名 | 型別 | 必填 | 說明 | 例子 |
| --- | --- | --- | --- | --- |
| loans.checked_out_at | datetime | Y | 借出時間（用於期間統計） | `2025-12-01T08:10:00Z` |
| loans.returned_at | datetime | N | 歸還時間（計算周轉） | `2025-12-10T10:00:00Z` |
| loans.status | enum | Y | 開放/關閉/逾期 | `overdue` |
| bibliographic_records.classification | string | N | 分類（做分類統計） | `823.914` |
| bibliographic_records.subjects | string[] | N | 主題（主題統計） | `["小說"]` |
| users.role | enum | Y | 角色（學生/教師） | `student` |
| users.org_unit | string | N | 班級/單位（分群） | `501` |
| item_copies.status | enum | Y | 遺失/報廢等異常 | `lost` |

## 6) 對應到系統的功能設計建議（至少 8 條）
- 需求：快速掌握利用情況 → 功能：熱門書 Top N（期間可選）→ 為何：支援閱讀推廣與補書。
- 需求：老師需要班級回饋 → 功能：班級借閱量（年級/班級彙總）→ 為何：作為閱讀課與活動成效指標。
- 需求：館藏發展要有依據 → 功能：分類借閱量 + 零借閱清單 → 為何：支援採購與汰舊決策。
- 需求：逾期要可處理 → 功能：逾期清單（含聯絡資訊由校內流程決定）→ 為何：降低館藏流失。
- 需求：找出資料問題 → 功能：異常偵測（某分類長期 0 借閱、條碼重複、狀態不一致）→ 為何：資料品質直接影響服務。
- 需求：分析但不侵入 → 功能：搜尋/瀏覽事件只存彙總（不存可識別個人）→ 為何：降低個資風險。
- 需求：外部可再利用 → 功能：所有報表可匯出 CSV，欄位穩定 → 為何：避免被單一系統綁死。
- 需求：學校要交代成效 → 功能：學期報告模板（自動帶入核心指標）→ 為何：減少行政負擔。

## 7) 常見坑與避雷清單（至少 8 條）
- 指標口徑不一致（不同人拉報表得到不同數字）。
- 報表過多但沒人用，反而增加維運成本。
- 以個人借閱排行做績效，造成不必要的標籤化與壓力。
- 為了分析保留完整明細與長期歷史，提升個資外洩風險。
- 分群過細（班級+性別+稀有行為）可能再識別到特定學生。
- 只做儀表板不做匯出，資料無法交付研究或校務需求。
- 沒有資料品質檢查，錯分/錯架導致「看起來沒人借」的假象。
- 把「零借閱」直接等同汰除，忽略參考書、典藏、課程用書。

## 8) 可交付給 AI 小助理的任務清單（至少 10 條）
1. 定義 MVP 報表清單與每份報表的欄位、篩選、計算口徑。
2. 產出「學期閱讀報告」模板（校內可直接使用）。
3. 設計零借閱清單的判定規則（期間、排除類型、例外處理）。
4. 建立分類借閱統計的呈現方式（主類/分區/年級對照）。
5. 設計異常偵測規則（條碼重複、狀態不一致、可疑 0 借閱）。
6. 規劃「搜尋/瀏覽」彙總事件的最小紀錄方案（避免存個資）。
7. 產出匯出 CSV 的欄位標準與版本管理策略。
8. 撰寫報表使用指南（每份報表回答什麼問題、怎麼解讀）。
9. 規劃閱讀推廣活動的成效指標（前後測、熱門主題變化）。
10. 提出 UI 的瀏覽入口設計（年級友善：熱門、主題區、推薦清單）。
